## Domain-First Development Policy

Japhathakam follows a Domain-First Development approach.

Every new Tritha Siddhantha concept shall be:

1. Discovered and discussed.
2. Defined in the Tritha Siddhantha Domain Dictionary.
3. Reviewed and approved.
4. Reflected in the software architecture.
5. Implemented in code.

Implementation follows the domain; the domain never follows the implementation.

If implementation conflicts with the approved Domain Dictionary, the implementation shall be revised unless an Architecture Decision Record (ADR) formally updates the domain definition.
